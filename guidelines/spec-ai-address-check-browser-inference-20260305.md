# AI住所チェック GitHub Pages対応（ブラウザ内推論）仕様（2026-03-05）

## 背景
- GitHub Pages は HTTPS 配信のため、HTTP のローカルLLM（Ollama）へ直接接続できない。
- 公開ページでもAI住所チェックを利用できる構成へ切り替える必要がある。

## スコープ
- AI住所チェックを、外部HTTP呼び出し方式からブラウザ内推論方式へ変更する。
- 推論は入力住所と KEN_ALL 由来候補の一致度スコアで実装する。
- 修正候補は候補スコア上位から1件提示する。
- 既存のAIチェックUI（判定/理由/信頼度/候補適用）は維持する。

## 制約
- サーバーや有料APIを追加しない。
- GitHub Pages（静的配信）で動作する。
- 住民票モードの既存機能へ影響を与えない。

## 受け入れ条件
- GitHub Pages上で `AIチェック` 実行時に `Failed to fetch` にならない。
- 判定結果と理由が表示される。
- 修正候補がある場合、`候補を適用` で住所欄へ反映できる。

## 非スコープ
- 高精度な大規模生成モデル推論の導入。
- サーバーサイド中継APIの新設。
