# 基本モード AI住所チェック（ローカルLLM）仕様（2026-03-05）

## 背景
- 基本モード入力中に、住所が実在するかをその場で確認したい。
- 間違いが疑われる場合に、修正候補住所を提示したい。
- 外部有料APIではなく、ローカルLLMで動作させたい。

## スコープ
- 基本モードの住所入力エリア（都道府県/市区町村/町域付近）に `AIチェック` を追加する。
- `AIチェック` 実行時、現在入力値をローカルLLMへ送信し、以下を取得する。
  - 実在妥当性（真偽）
  - 判定理由
  - 修正候補住所（任意）
  - 信頼度（0〜1）
- 結果をUI表示し、修正候補がある場合は入力欄へ反映できる。
- ローカルLLM設定（エンドポイント、モデル名）を設定画面から変更できる。
- `.env` キーから初期値を読み込める。

## 制約
- ローカルLLMはブラウザから直接アクセス可能なHTTPエンドポイントを前提とする。
- 既存の住所補完、保存、シート書き込みの挙動は変更しない。
- 住所候補は既存KEN_ALLデータを活用し、LLMへ補助情報として渡す。
- ローカルLLM設定が空欄または接続失敗時は、ユーザーへ明示的なエラーを表示する。

## 受け入れ条件
- 基本モードで `AIチェック` ボタンが表示され、押下で判定が開始される。
- 判定完了後、妥当性・理由・信頼度が表示される。
- 修正候補がある場合、`候補を適用` で都道府県/市区町村/町域/郵便番号へ反映できる。
- 設定画面でローカルLLMエンドポイント/モデルを変更できる。
- `.env` から `VITE_LOCAL_LLM_ENDPOINT` / `VITE_LOCAL_LLM_MODEL` が読める。

## 非スコープ
- 住民票モードへの同機能追加。
- ローカルLLMサーバー自体のセットアップ自動化。
- サーバーサイド中継APIの新設。
