# 仕様書: 基本モード書き込み除外チェック + 電話番号先頭0自動補完（2026-03-06）

## 背景
- 基本モードで住所系の一部項目は入力値を保持したまま、シート書き込み時だけ除外したい。
- 携帯番号入力時に `09` / `08` / `07` で止まった場合、次の `0` を自動補完して入力効率を上げたい。

## スコープ
1. 基本モードの以下項目に「書き込み除外」チェックボックスを追加する。
- 郵便番号
- 都道府県
- 市区町村
- 町域

2. チェックされた項目は、入力欄に値があっても基本モードのシート書き込み payload では空文字として送る。

3. 上記チェック状態は、以下操作でリセットしない。
- クリア
- リスト保存後の自動クリア
- 書き込み後の自動クリア

4. 電話番号入力（基本モード）で、携帯入力モード時に `09` / `08` / `07` まで入力された時点で、3桁目の `0` を自動補完する。

## 制約
- 既存のUI配置を大きく崩さない。
- 既存の電話番号整形仕様（0289固定電話パターン、携帯/固定切替）を維持する。
- 追加機能は既存のローカル保存（リロード復元）と整合する。

## 受け入れ基準
- 基本モードUIで対象4項目にチェックボックスが表示される。
- 例えば「都道府県」にチェックした状態で書き込みすると、都道府県入力値はシート側住所連結に含まれない。
- チェック状態は `クリア` 実行後も維持される。
- `09` と入力した時点で `090` に自動補完される（`08`->`080`, `07`->`070` も同様）。

## 非スコープ
- 住民票モードへの同等チェックボックス追加
- Google Apps Script 側の列構成変更
- 電話番号の全パターン再設計
