# 列空セル優先書き込みとリスト個別上書き仕様（2026-03-04）

## 対象
- `src/app/components/DataEntryForm.tsx`
- `guidelines/resident-sheet-webhook-setup-20260303.md`
- `package.json`
- `package-lock.json`

## スコープ
- 書き込み時は、対象列が空の先頭行へ反映する。
- 住民票シート2（B/C列）では、氏名（C列）の空セルを優先して上から埋める。
- 住民票シート2のリスト各項目に、1件ずつ実行できる `上書き書き込み` ボタンを追加する。
- 住民票通常書き込み（B/F/G/H/I/J/K/L）も、対象列が空の先頭行を優先する。
- 基本モード通常書き込み（A:J）も、対象列が空の先頭行を優先する。

## 制約
- 既存アクション名は維持する。
- 既存Webhookレスポンスの主要キー（`ok` / `row` / `rowsWritten` / `startRow` / `endRow` / `sheetName`）は維持する。
- 既存のUI構成・モード切替は壊さない。

## 受け入れ条件
- 住民票シート2で、既存データが他列にあっても、C列の最上部空セルから書き込まれる。
- 住民票シート2リストの各行に `上書き書き込み` ボタンが表示される。
- `上書き書き込み` 実行時、対象1件だけがWebhook送信される。
- 基本モード通常書き込みが `A:J` の最上部空行へ反映される。
- 住民票通常書き込みが `B/F/G/H/I/J/K/L` の最上部空行へ反映される。

## 非対象
- シート初期化範囲の変更
- 列マッピング自体の変更
- Webhook URLや認証方式の変更
