# ホームページFigma化と補完表示改善仕様（2026-03-03）

## 対象
- `src/app/components/DataEntryForm.tsx`
- `src/app/lib/addressSuggestions.ts`
- `src/app/lib/prefectureSearch.ts`

## スコープ
- Figma MCP を用いて現在のホームページ（本アプリ画面）を Figma ファイルとして生成する。
- 補完表示を「常時表示」から「入力フォーカス中のみ表示」に変更する。
- 補完候補の表示テキストを全角空白区切りに統一する。
- 件数上限や単一カテゴリ優先表示を廃止し、該当候補をすべて表示する。
- ローマ字入力時（例: `sa`）は、候補のうちローマ字が入力値で始まるものを上位表示する（都道府県・市区町村・町域）。
- 候補表示は各入力欄直下のドロップダウンメニュー形式で表示する。
- 町域ドロップダウンでは郵便番号（数字列）を表示しない。
- 漢字入力は先頭1文字から候補を表示する（都道府県・市区町村・町域）。
- 候補表示中は `↓` / `↑` キーで候補移動し、`Enter` キーで選択できる。
- ローマ字ゆらぎ（例: `shi` / `si`）のどちらでも同一候補を検索できる。
- 町域入力のIME変換中は候補計算を停止し、入力遅延を抑える。

## 制約
- 既存の保存・削除・コピー・PDF/シート切替など基本機能を壊さない。
- サーバー追加は行わず、既存のフロントエンド構成を維持する。
- 住所マスタ読込中・読込失敗・候補なし表示は維持する。

## 受け入れ条件
- 郵便番号・住所欄の補完候補は、対象入力エリアにフォーカスがあるときのみ表示される。
- 候補クリック後は補完パネルが閉じる。
- 候補文字列は各要素間が全角空白（`　`）で表示される。
- 都道府県候補、市区町村候補、町域候補がある場合、該当カテゴリをすべて表示する。
- 候補件数は任意の該当件数まで表示される（固定上限で切り捨てない）。
- 都道府県・市区町村・町域の各候補で、`sa` 入力時は `saitama` のような `sa` 始まり候補が上位に並ぶ。
- 郵便番号・都道府県・市区町村・町域の候補が、該当入力欄の直下にドロップダウンとして表示される。
- 町域ドロップダウン候補は `都道府県　市区町村　町域` のみ表示し、郵便番号は表示しない。
- 漢字1文字入力（例: `北`）でも該当候補が表示される。
- 候補表示中に `↓` / `↑` / `Enter` でキーボード選択できる。
- `shi` / `si` のような入力差異で同一候補がヒットする。
- 町域の日本語入力中（IME変換中）は体感遅延が軽減される。
- `npm run build` が成功する。

## 非対象
- 住民票モード（転出・本籍）の補完未実装箇所の新規実装。
- 補完検索アルゴリズム自体（マッチング規則）の全面変更。
