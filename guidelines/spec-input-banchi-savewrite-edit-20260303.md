# 入力書式・番地入力・保存導線改善仕様（2026-03-03）

## 対象
- `src/app/components/DataEntryForm.tsx`
- `src/styles/theme.css`
- `guidelines/spec-input-banchi-savewrite-edit-20260303.md`
- `package.json`
- `package-lock.json`

## スコープ
- シート書き込み時のセル書式をメイリオ、サイズ10に統一する。
- 番地入力で英字・数字・ハイフン・漢字を入力可能にする。
- 番地入力では英字・数字を強制的に全角へ正規化する。
- 住民票モードの保存導線を分離する。
  - `リストへ保存`: リスト保存のみ実施し、実行後に入力欄をクリアする。
  - `書き込み`: シート反映のみ実施する。
- 住民票モードのボタン配置を `リストへ保存（左）`、`書き込み（右）` にする。
- 保存済みリストの操作に編集ボタンを追加する（削除ボタンの右）。

## 制約
- 既存のシート列マッピング（B/F/G/H/I/J/K/L）と開始行6は維持する。
- 既存の住民票シート選択・タブ保持ロジックを壊さない。
- サーバー常駐追加は行わない。

## 受け入れ条件
- シート書き込み時の対象セルがメイリオ10で記録される。
- 番地欄で英字/数字/漢字/「－」を入力できる。
- 番地欄に英字/数字を入力すると全角化される。
- 住民票モードで `リストへ保存` はシート反映せず、入力がクリアされる。
- 住民票モードで `書き込み` 実行時のみシート反映が走る。
- 保存済みリストに編集ボタンが表示され、押下で入力欄へ値を戻せる。
- `npm run build` が成功する。

## 非対象
- 基本モードへのシート書き込み機能追加
