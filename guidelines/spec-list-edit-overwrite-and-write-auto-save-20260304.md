# リスト編集上書き保存と書き込み時自動リスト保存仕様（2026-03-04）

## 対象
- `src/app/components/DataEntryForm.tsx`
- `package.json`
- `package-lock.json`

## スコープ
- 基本モードの保存済みリストで `編集` から入力欄へ戻したデータは、`リストへ保存` 時に新規追加ではなく同一IDへ上書き保存する。
- 住民票モード（住民票シート1）の保存済みリストも同様に、編集後の `リストへ保存` は同一IDへ上書き保存する。
- 基本モードの `書き込み` 成功時に、同じ内容をリストへ自動保存（編集状態なら上書き）する。
- 住民票モード（住民票シート1）の `書き込み` 成功時に、同じ内容をリストへ自動保存（編集状態なら上書き）する。

## 制約
- 住民票シート2の既存仕様（リスト保存を使わない）は維持する。
- 既存のシートWebhook送信ペイロード構造は変更しない。
- 既存の削除/編集ボタン機能は維持する。

## 受け入れ条件
- 編集対象を読み込んだ後に `リストへ保存` を押すと件数が増えず、対象データが更新される。
- 新規入力で `リストへ保存` を押すと従来通り件数が増える。
- `書き込み` 成功後にリストへ同データが反映される。
- 編集状態で `書き込み` 成功後、編集対象が上書き更新される。

## 非対象
- 住民票シート2のリスト運用追加
- シート書き込み列マッピング変更
