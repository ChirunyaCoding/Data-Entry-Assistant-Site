# リスト上書き時の元行優先書き込み仕様（2026-03-04）

## 対象
- `src/app/components/DataEntryForm.tsx`
- `guidelines/resident-sheet-webhook-setup-20260303.md`
- `package.json`
- `package-lock.json`

## スコープ
- 基本モードの保存済みリスト `上書き書き込み` は、可能な限り元の行へ上書きする。
- 住民票モード（住民票シート1）の保存済みリスト `上書き書き込み` は、可能な限り元の行へ上書きする。
- リスト項目に、シート別の最終書き込み行を保持し、次回上書き時に再利用する。
- Webhook（Apps Script）側は、`targetRow` と `preferExistingRow` を受け取れるようにする。

## 制約
- 通常の `書き込み`（新規追記）挙動は維持する。
- 住民票シート2（B/C列）の既存ロジックは変更しない。
- 既存のレスポンス形式（`ok` / `row` / `rowsWritten` / `startRow` / `endRow` / `sheetName`）は維持する。

## 受け入れ条件
- 同一リスト項目で `上書き書き込み` を繰り返しても、行が増えず同一行が更新される。
- 既存項目で行番号未保持の場合も、`preferExistingRow` で一致行が見つかればその行へ上書きされる。
- 既存一致行が見つからない場合のみ、従来どおり空行へ書き込み、成功時に行番号を記録する。

## 非対象
- シート全体の重複解消
- 並び順永続化ロジックの変更
