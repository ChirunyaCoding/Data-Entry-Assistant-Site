# 住民票モード自分の名前固定化仕様（2026-03-03）

## 対象
- `src/app/components/DataEntryForm.tsx`
- `guidelines/spec-resident-self-name-fixed-20260303.md`
- `package.json`
- `package-lock.json`

## スコープ
- 設定モーダルに「自分の名前を固定する」設定を追加する。
- 住民票モードの「自分の名前（B列）」入力を、固定有効時は設定値で表示し編集不可にする。
- 固定値はローカルストレージへ保存し、再読み込み後も復元する。
- 住民票モード保存時・クリア時に固定値が反映されるようにする。

## 制約
- 既存の「入力者名を固定する」設定挙動を変更しない。
- 既存の住民票モード保存、コピー、シート連携の列マッピングを変更しない。
- 固定値はハードコードせず、既存設定ストレージに統合する。

## 受け入れ条件
- 設定モーダルに以下が表示される。
  - `自分の名前を固定する` チェックボックス
  - `固定する自分の名前` 入力欄
- 固定ON時、住民票モードの`自分の名前（B列）`欄がグレーアウトし編集不可になる。
- 固定ON時、保存データのB列値に固定名が入る。
- 固定ON時、クリア実行後も`自分の名前`は固定名のまま維持される。
- `npm run build` が成功する。

## 非対象
- Google Apps Script 側の実装変更
- 基本モード側の新規UI追加
