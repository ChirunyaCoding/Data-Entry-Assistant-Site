# 住民票モードシート保存連携仕様（2026-03-03）

## 対象
- `src/app/components/DataEntryForm.tsx`
- `guidelines/spec-resident-sheet-column-mapping-20260303.md`
- `package.json`
- `package-lock.json`

## スコープ
- 住民票モードの「保存」で、ローカル保存に加えてシート反映用の送信処理を実行する。
- シート列マッピングを以下に固定する。
  - B列: 自分の名前
  - F列: 本人氏名（転出）
  - G列: 住所（転出）
  - H列: 建物名（転出）
  - I列: 本人氏名（本籍）
  - J列: 住所（本籍）
  - K列: 建物名（本籍）
  - L列: 通称・別名
- 住民票モードに `通称・別名` 入力欄を追加する（本籍建物名の直後）。
- 住民票モードで `registryBuilding` から `residentAlias` への移動は `Shift+Enter` のときのみ許可する。
- シート反映データは 6行目以降へ追記する前提で送信する。

## 制約
- 既存の住民票モード入力・補完・保存済み一覧・コピー機能を壊さない。
- サーバー常駐は追加せず、フロントエンドから既存連携先（Webhook）へ送信する。
- 機密情報はハードコードせず、接続先URLは環境変数で扱う。

## 受け入れ条件
- 住民票モードに「自分の名前」「通称・別名」入力欄が表示される。
- 本籍建物名欄で Enter のみ押下時は通称・別名欄へ移動しない。
- 本籍建物名欄で Shift+Enter 押下時のみ通称・別名欄へ移動する。
- 住民票モード保存時、列マッピングに沿ったデータをシート送信用に構築する。
- 行開始位置 6 を含む送信ペイロードを送る。
- `npm run build` が成功する。

## 非対象
- Google Apps Script 側ロジック自体のデプロイ作業。
- 基本モードのシート保存連携。
