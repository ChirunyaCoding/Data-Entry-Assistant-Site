# 住民票シート1 フォルダ読み込み書き込み仕様（2026-03-03）

## 対象
- `src/app/components/DataEntryForm.tsx`
- `guidelines/resident-sheet-webhook-setup-20260303.md`
- `guidelines/spec-resident-sheet1-folder-import-20260303.md`
- `package.json`
- `package-lock.json`

## スコープ
- 住民票モードに「フォルダを読み込み」ボタンを追加する。
- 選択フォルダ内の子フォルダ配下にある `tif/tiff` ファイルを抽出し、住民票シート1向けに一括書き込みできるようにする。
- 書き込み列は `C/D/E` 固定とし、`6` 行目以降へ上から順に追記する。
  - `C列`: 選択フォルダ名
  - `D列`: 子フォルダ名（対象 `tif/tiff` 件数分だけ繰り返し）
  - `E列`: `tif/tiff` ファイル名（拡張子付き）

## 制約
- 住民票シート1（`residentPrimary`）でのみ実行可能にする。
- 書き込み先は既存の「書き込み先シート」選択値を利用する。
- サーバー常駐は追加しない。既存Apps Script Webhookを拡張して利用する。
- 既存の住民票通常書き込み（`B/F/G/H/I/J/K/L`）仕様は変更しない。

## 受け入れ条件
- 住民票モードで「フォルダを読み込み」ボタンが表示される。
- 住民票シート1選択時のみ実行できる。
- フォルダ選択後、子フォルダ内の `tif/tiff` 一覧に基づいて `C/D/E` が `6` 行目以降へ追記される。
- `tif/tiff` が1件もない場合にエラーメッセージが表示される。
- `npm run build` が成功する。

## 非対象
- 既存フォーム入力項目の構成変更
- 住民票シート2向けの同機能追加
- フォルダ構造の再帰的な深い階層マッピング（即時子フォルダ以外の階層構造保存）
