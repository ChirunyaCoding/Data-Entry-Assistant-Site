# 住民票反映先シートタブ選択仕様（2026-03-03）

## 対象
- `src/app/components/DataEntryForm.tsx`
- `guidelines/spec-resident-target-sheet-tab-20260303.md`
- `package.json`
- `package-lock.json`

## スコープ
- 住民票モード保存時の反映先を、同一スプレッドシート内のシートタブ名で選択できるようにする。
- 反映ペイロードへ `sheetName` を追加する。
- 住民票モード画面に「書き込み先シート名」入力欄を追加し、保存先タブ名を指定可能にする。
- 指定したシートタブ名を `localStorage` へ保持し、再訪時に復元する。
- シートタブ名未入力時は保存処理でエラー表示する。

## 制約
- 既存の列マッピング（B/F/G/H/I/J/K/L）と開始行6は維持する。
- 既存の住民票データ入力と保存済みリスト表示を壊さない。
- サーバー追加は行わず、既存Webhook連携を利用する。

## 受け入れ条件
- 住民票モードで書き込み先シート名を指定して保存できる。
- 送信ペイロードに `sheetName` が含まれる。
- 書き込み先シート名未入力時に保存エラーが表示される。
- `npm run build` が成功する。

## 非対象
- Google Apps Script側のタブ選択ロジックの自動更新。
- 基本モードの書き込み機能。
