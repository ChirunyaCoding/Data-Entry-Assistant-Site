# 設定画面への.env読込追加仕様（2026-03-04）

## 対象
- `src/app/components/DataEntryForm.tsx`
- `package.json`
- `package-lock.json`

## スコープ
- 設定画面に、シートURL/Webhook URLの設定項目を追加する。
- URL設定の初期値は `.env`（`import.meta.env`）から読み込む。
- 既存の設定（入力者固定・自分の名前固定）と同じ `localStorage` へ保存し、再読み込み後も維持する。
- 設定画面に「`.env` の値を読み込む」操作を追加し、URL設定を `.env` 初期値へ戻せるようにする。
- シート表示・シートタブ取得・書き込み先判定・Webhook送信先が、設定値（未設定時は `.env` 初期値）を参照するようにする。

## 制約
- 既存の書き込みロジック（空行探索や上書き処理）の仕様は変更しない。
- 有料サービス・サーバー機能は追加しない。
- 既存UIの構成変更は設定モーダル内に限定する。

## 受け入れ条件
- `.env` が設定済みなら、設定画面のURL項目に初期表示される。
- URL項目を変更すると、その値がシート表示/書き込み/Webhook送信に反映される。
- 「`.env` の値を読み込む」を押すとURL項目が `.env` 値に戻る。
- `npm run build` が成功する。

## 非対象
- `.env` ファイル自体をブラウザ実行時に直接読み取る機能（ファイルIO）
- 既存のシートデータ構造変更
