# シート初期化ボタンと基本モード書き込み仕様（2026-03-03）

## 対象
- `src/app/components/DataEntryForm.tsx`
- `guidelines/resident-sheet-webhook-setup-20260303.md`
- `guidelines/spec-sheet-initialize-and-basic-write-20260303.md`
- `package.json`
- `package-lock.json`

## スコープ
- シート選択ドロップダウンの右に `初期化` ボタンを追加する。
- `初期化` はモードと対象シートに応じて以下の範囲を一括クリアする。
  - 基本モード: 全列 `5` 行目から `1000` 行目
  - 住民票モード（住民票シート1）: 全列 `6` 行目から `1000` 行目
  - 住民票モード（住民票シート2）: `B3:C1000`
- 基本モードに `書き込み` ボタンを追加し、選択中タブへ `5` 行目以降へ追記する。
- 基本モードの列マッピングを以下で実装する。
  - A: 入力者名
  - B: ファイル名
  - C: 郵便番号
  - D: 都道府県 + 市区町村 + 町域 + 大字 + 字 + 小字 + 番地（結合）
  - E: 建物名
  - F: 会社名
  - G: 肩書
  - H: 氏名
  - I: 電話番号
  - J: 備考
- 基本モードのボタン仕様を住民票モードと同様にする。
  - `リストへ保存`: リスト保存のみ、実行後に入力クリア
  - `書き込み`: シート反映のみ

## 制約
- 既存の住民票モード列マッピング（B/F/G/H/I/J/K/L）と開始行6は維持する。
- 既存のシートタブ一覧取得・保持ロジックを壊さない。
- サーバー常駐追加は行わず、既存Apps Script Webhookを拡張する。

## 受け入れ条件
- シート選択UIの右側に `初期化` ボタンが表示される。
- `初期化` 実行時、対象範囲の値が一括削除される。
- 基本モードで `書き込み` 実行時、指定マッピングで5行目以降へ追記される。
- 基本モードの `リストへ保存` 実行時、入力がクリアされる。
- `npm run build` が成功する。

## 非対象
- Googleスプレッドシートの共有権限設定変更
- 基本モード以外の列マッピング変更（住民票既存分を除く）
