# シートタブ選択化と表示復元仕様（2026-03-03）

## 対象
- `src/app/components/DataEntryForm.tsx`
- `guidelines/spec-sheet-tab-selector-and-persist-20260303.md`
- `guidelines/resident-sheet-webhook-setup-20260303.md`
- `package.json`
- `package-lock.json`

## スコープ
- 住民票モードの「書き込み先シート名」を手入力から選択式に変更する。
- 固定URLのスプレッドシート内にあるシートタブ一覧を取得し、選択候補として表示する。
- シート表示（右ペイン）でもタブを選択できるようにし、選択中タブをURL `gid` に反映する。
- PDF/シート/漢字切替、基本/住民票モード切替、住民票シート1/2切替を行っても、各スプレッドシートで最後に選んだタブを保持する。

## 制約
- 既存の固定スプレッドシートURL（基本1つ、住民票2つ）は維持する。
- 住民票保存の列マッピング（B/F/G/H/I/J/K/L）と開始行6は維持する。
- サーバー常駐は追加しない。既存のApps Script Webhookを拡張して利用する。
- シークレット値はコードへハードコードしない。

## 受け入れ条件
- 住民票モードで書き込み先を `select` から選択でき、手入力欄は存在しない。
- シートビューで現在のスプレッドシート内タブを選択できる。
- 選択タブに応じて `iframe` の表示先が変わる（`gid` 反映）。
- モードやビューを行き来しても、各スプレッドシートごとの選択タブが維持される。
- 住民票保存時、選択中タブ名が `sheetName` として送信される。
- `npm run build` が成功する。

## 非対象
- Googleスプレッドシートの共有権限設定変更
- 基本モードへの保存機能追加
